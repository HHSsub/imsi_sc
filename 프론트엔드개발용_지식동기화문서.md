## 🎯 R&R (Role & Responsibility)

### 역할 (Role)
React 기반 사용자 인터페이스 구현 전담

### 책임 (Responsibility)

#### 1. UI 컴포넌트 개발
- **대시보드 홈**: KPI 카드 (총 리뷰수, 평균 별점, 긍정/부정 비율)
- **시계열 차트**: Chart.js로 긍정/부정 리뷰 추이 시각화
- **키워드 리스트**: 긍정/부정 키워드 Top 10 표시
- **공통 컴포넌트**: Header, Footer, 로딩 스피너 등

#### 2. API 연동
- Axios 또는 Fetch API로 백엔드 호출
- 응답 데이터 파싱 및 상태 관리
- 에러 핸들링 (네트워크 오류, 404, 500 등)
- 로딩 상태 표시

#### 3. 스타일링
- Tailwind CSS 기반 디자인 시스템 구현
- PRD의 컬러 팔레트 및 타이포그래피 준수
- 반응형 레이아웃 (모바일 대응은 추후)

#### 4. 라우팅 (추후)
- React Router로 페이지 전환 구현 (v1.0에서는 단일 페이지만)

## 🔌 API 계약서

**Base URL**: `http://localhost:8000` (기본값)

### 환경변수
- `VITE_API_BASE_URL`: API Base URL (기본값: `http://localhost:8000`)
- `VITE_USE_MOCK`: `true|false` (기본값: `true`, mock 기반으로 UI/UX 먼저 구현)

프론트엔드는 다음 API 엔드포인트를 호출합니다 (백엔드와 합의):

### 0. 챗봇 (핵심)

**엔드포인트**: `POST /api/chat`

**요청 스키마(초안)**:
```json
{
  "message": "사용자 질문",
  "history": [{"role":"user","content":"..."}, {"role":"assistant","content":"..."}]
}
```

**응답 스키마(초안)**:
```json
{
  "content": "AI 답변 텍스트",
  "products": [
    {
      "id": "string",
      "name": "string",
      "type": "string",
      "priceText": "string",
      "avgRating": "string|number",
      "totalReviews": 0,
      "trustScore": 0,
      "highlight": "string"
    }
  ]
}
```

### 0-1. 제품 비교 (v2.0)

**엔드포인트**: `POST /api/compare`

**요청 스키마(초안)**:
```json
{
  "products": ["productIdA", "productIdB"]
}
```

**응답 스키마(초안)**:
```json
{
  "summary": "AI 비교 요약",
  "rows": [
    { "category": "보습력", "aText": "⭐⭐⭐⭐ 78%", "bText": "⭐⭐⭐⭐⭐ 92%", "winner": "a|b|tie" }
  ]
}
```

### 0-2. 제품 상세 (v2.0)

**엔드포인트**: `GET /api/products/{id}`

**응답 스키마(초안)**:
```json
{
  "product": { "id": "string", "name": "string" },
  "summary": "AI 요약",
  "categoryRatings": [{ "category": "보습력", "scoreText": "⭐⭐⭐⭐", "ratioText": "78% 만족" }],
  "trustedReviews": [{ "id":"string", "rating":5, "trustScore":95, "date":"YYYY-MM-DD", "text":"..." }]
}
```

### 에러 처리

| 상태 코드 | 의미 | 처리 방법 |
|----------|------|----------|
| 200 | 성공 | 데이터 렌더링 |
| 404 | 제품명 없음 | "제품을 찾을 수 없습니다" 메시지 표시 |
| 500 | 서버 오류 | "서버 오류가 발생했습니다" 메시지 표시 |
| Network Error | 네트워크 오류 | "네트워크 연결을 확인해주세요" 메시지 표시 |

## 📌 주의사항 및 가이드라인

### 1. API 호출 시 주의사항
- **Base URL 환경 변수화**: `import.meta.env.VITE_API_BASE_URL` 사용
- **에러 핸들링 필수**: try-catch로 감싸고 사용자에게 친화적인 메시지 표시
- **로딩 상태 표시**: API 호출 중 로딩 스피너 표시

**예시**:
```javascript
// src/services/api.js
import axios from 'axios';

const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000';

export const fetchKPI = async (productName) => {
  try {
    const response = await axios.get(`${API_BASE_URL}/api/kpi?product=${productName}`);
    return response.data;
  } catch (error) {
    if (error.response?.status === 404) {
      throw new Error('제품을 찾을 수 없습니다');
    } else if (error.response?.status === 500) {
      throw new Error('서버 오류가 발생했습니다');
    } else {
      throw new Error('네트워크 연결을 확인해주세요');
    }
  }
};
```

```jsx
  return (
    <div>

        <span key={kw.word} style={{ color }}>
          {kw.word} ({kw.count})
        </span>
      ))}
    </div>
  );
}
```

## 🔍 예상 이슈 및 해결 방법

### 이슈 1: CORS 오류
**증상**: `Access-Control-Allow-Origin` 에러

**해결**: 백엔드에서 CORS 설정 필요 (Backend Agent에게 요청)
```python
# backend/app/main.py
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

---

### 이슈 2: Chart.js 렌더링 안됨
**증상**: 차트가 화면에 표시되지 않음

**해결**:
1. `react-chartjs-2` 및 `chart.js` 버전 확인
2. Chart 컴포넌트에 `width`, `height` props 명시
3. 데이터 형식 확인 (배열인지)

---

### 이슈 3: 제품명 한글 인코딩
**증상**: 제품명에 한글이 포함되어 API 호출 실패

**해결**: `encodeURIComponent` 사용
```javascript
const response = await axios.get(
  `${API_BASE_URL}/api/kpi?product=${encodeURIComponent('라운드랩')}`
);
```
