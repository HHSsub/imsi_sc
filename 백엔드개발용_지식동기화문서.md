- 시계열 집계 (groupby date)

- 프론트엔드 (localhost:5173) 접근 허용

## 🔌 API 엔드포인트 명세

**Base URL**: `http://localhost:8000`

### 1. KPI 데이터

**엔드포인트**: `GET /api/kpi`

**Query Parameters**:
- `product` (required): 제품명 (예: "라운드랩", "아누아")

**요청 예시**:
```
GET http://localhost:8000/api/kpi?product=라운드랩
```

**응답 스키마**:
```json
{
  "total_reviews": 7305,
  "avg_rating": 4.2,
  "positive_ratio": 0.78,
  "negative_ratio": 0.12
}
```

**응답 필드**:
- `total_reviews` (int): 총 리뷰 수
- `avg_rating` (float): 평균 별점 (1.0 ~ 5.0)
- `positive_ratio` (float): 긍정 리뷰 비율 (0.0 ~ 1.0)
- `negative_ratio` (float): 부정 리뷰 비율 (0.0 ~ 1.0)

**에러 응답**:
- `404`: 제품을 찾을 수 없음
  ```json
  {"detail": "Product not found"}
  ```
- `500`: 서버 오류
  ```json
  {"detail": "Internal server error"}
  ```

---

### 2. 시계열 데이터

**엔드포인트**: `GET /api/timeseries`

**Query Parameters**:
- `product` (required): 제품명

**요청 예시**:
```
GET http://localhost:8000/api/timeseries?product=라운드랩
```

**응답 스키마**:
```json
{
  "dates": ["2025-08-01", "2025-08-02", "2025-08-03", ...],
  "positive_counts": [45, 52, 38, ...],
  "negative_counts": [5, 8, 3, ...],
  "avg_ratings": [4.3, 4.5, 4.2, ...]
}
```

**응답 필드**:
- `dates` (list[str]): 날짜 배열 (ISO 8601 형식: YYYY-MM-DD)
- `positive_counts` (list[int]): 날짜별 긍정 리뷰 수
- `negative_counts` (list[int]): 날짜별 부정 리뷰 수
- `avg_ratings` (list[float]): 날짜별 평균 별점

**데이터 처리 로직**:
1. DB에서 해당 제품의 리뷰 조회
2. 날짜별 groupby
3. 별점 4-5점: 긍정, 1-2점: 부정으로 분류
4. 집계 후 정렬 (날짜 오름차순)

### 3. 긍정 키워드

**엔드포인트**: `GET /api/keywords/positive`

**Query Parameters**:
- `product` (required): 제품명
- `limit` (optional): 반환할 키워드 수 (기본값: 30)

**요청 예시**:
```
GET http://localhost:8000/api/keywords/positive?product=라운드랩&limit=10
```

**응답 스키마**:
```json
{
  "keywords": [
    {"word": "촉촉", "count": 1234},
    {"word": "보습", "count": 987},
    {"word": "효과", "count": 876},
    ...
  ]
}
```

**응답 필드**:
- `keywords` (list[dict]): 키워드 배열
  - `word` (str): 키워드
  - `count` (int): 빈도

**데이터 처리 로직**:
1. 별점 4-5점인 리뷰만 필터링
2. `clean_text` 또는 `raw_text`에서 키워드 추출
3. 정규표현식 또는 단순 단어 분리
4. 빈도 계산 및 정렬 (내림차순)
5. 상위 `limit` 개 반환

```python
# backend/app/db/init_db.py
import pandas as pd
from sqlalchemy.orm import Session
from .database import engine, SessionLocal
from .models import Review

def load_csv_to_db(csv_path: str, product_name: str):
    df = pd.read_csv(csv_path, encoding='utf-8-sig')

    session = SessionLocal()
    try:
        for _, row in df.iterrows():
            review = Review(
                product_name=product_name,
                product_type=row.get('제품타입', ''),
                price=row.get('가격', 0),
                date=row.get('날짜', ''),
                rating=row.get('별점', 0),
                skin_type=row.get('스킨타입', ''),
                tags=row.get('태그', ''),
                raw_text=row.get('리뷰내용', ''),
                # clean_text는 추후 전처리 후 업데이트
            )
            session.add(review)
        session.commit()
    finally:
        session.close()

if __name__ == "__main__":
    load_csv_to_db('../data/roundlab_comprehensive_reviews.csv', '라운드랩')
    load_csv_to_db('../data/final_anua_reviews.csv', '아누아')
```

### 4. 키워드 추출 로직 (간단 버전)

프로토타입에서는 정규표현식 기반 단순 추출:

```python
# backend/app/services/keyword_service.py
import re
from collections import Counter

def extract_keywords(texts: list[str], limit: int = 30):
    """
    간단한 키워드 추출 (정규표현식 기반)
    """
    # 모든 텍스트 합치기
    combined_text = ' '.join(texts)

    # 한글 단어만 추출 (2글자 이상)
    words = re.findall(r'[가-힣]{2,}', combined_text)

    # 불용어 제거 (기본적인 것만)
    stopwords = ['이거', '정말', '진짜', '완전', '너무', '엄청', '약간', '조금']
    words = [w for w in words if w not in stopwords]

    # 빈도 계산
    word_counts = Counter(words)

    # 상위 limit 개 반환
    top_words = word_counts.most_common(limit)

    return [{"word": word, "count": count} for word, count in top_words]
```

*추후 형태소 분석기 (KoNLPy Okt/Mecab) 도입 고려*

### 이슈 4: 날짜 형식 불일치
**증상**: CSV의 날짜가 다양한 형식 (2025.08.01, 2025-08-01 등)

**해결**:
```python
import pandas as pd

# 날짜 파싱 및 표준화
df['날짜'] = pd.to_datetime(df['날짜'], errors='coerce').dt.strftime('%Y-%m-%d')
```
