## 1. 서비스 개요 (AI 중심 재정의)

### 🎯 1.1 서비스 배경 및 목적

#### 배경
**소비자의 문제**: 화장품을 구매하기 전, 소비자들은 수백~수천 개의 리뷰를 읽으며 다음과 같은 어려움을 겪습니다:
- 😵 "어떤 제품이 내 피부에 맞을까?" - 피부타입별 정보 부족
- ⏰ "리뷰를 다 읽기엔 시간이 너무 오래 걸려" - 시간 소모적
- 🤔 "이 리뷰는 믿을 수 있는 건가?" - 신뢰도 판단 어려움
- 🆚 "A 제품과 B 제품 중 뭐가 더 나을까?" - 비교 분석 어려움

**기존 솔루션의 한계**:
- 별점만으로는 제품의 세부 특성(보습력, 자극도 등)을 파악하기 어려움
- 필터링 기능이 제한적 (별점, 최신순 정도)
- 자연어 질문 불가능 ("건성 피부에 좋은 세럼 추천해줘" 같은 질의 불가)

#### 목적
**30초 안에 나에게 맞는 제품을 찾아주는 AI 어드바이저**를 개발합니다.
- 🤖 **LLM Agent**가 30,000+ 리뷰를 학습하여 자연어 질의응답
- 📊 **RAG (Retrieval-Augmented Generation)**로 리뷰 기반 답변 생성
- 🎯 **피부타입별 맞춤 추천** 및 **제품 간 심층 비교**
- ⭐ **리뷰 신뢰도 평가**로 신뢰할 수 있는 정보 제공

### 👥 1.2 타겟 사용자 (B2C - 일반 소비자)

| 페르소나 | 니즈 | 사용 시나리오 |

## 2. AI/LLM 기술 스택

### 🤖 2.1 LLM (Large Language Model) - 유연한 모델 선택

**핵심 설계 원칙**: **모델 독립성** - 다양한 LLM을 쉽게 교체 가능하도록 추상화

#### 옵션 1: OpenAI GPT-4 (권장 - 프로토타입)
| 항목 | 상세 |
|------|------|
| **모델** | GPT-4 Turbo / GPT-4o |
| **장점** | 높은 품질, 한국어 성능 우수, API 사용 간편 |
| **단점** | 유료 (비용 발생), 외부 의존성 |
| **비용** | ~$0.01/1K tokens (입력), ~$0.03/1K tokens (출력) |
| **용도** | 자연어 질의응답, 제품 비교 요약, 추천 생성 |

#### 옵션 2: Self-Hosted Open-Source LLM (비용 절감)
| 모델 | 크기 | 특징 | GPU 요구사항 |
|------|------|------|-------------|
| **Llama 3 (8B)** | 8B params | Meta, 한국어 준수, 상업적 사용 가능 | 24GB VRAM (A5000, RTX 4090) |
| **Qwen 2.5 (7B)** | 7B params | Alibaba, 한국어 우수, 코딩 능력 강함 | 16GB VRAM |
| **Mistral 7B** | 7B params | 유럽 기반, 빠른 추론 속도 | 16GB VRAM |
| **EEVE-Korean 10.8B** | 10.8B params | 한국어 특화 (LLaMa 기반 fine-tuning) | 24GB VRAM |

**배포 방식**:
- **vLLM** (고속 추론 서버) 또는 **Ollama** (로컬 실행 간편)
- **Hugging Face TGI** (Text Generation Inference)

#### 옵션 3: Small Language Model (SLM) - 로컬 실행
| 모델 | 크기 | 특징 | 실행 환경 |
|------|------|------|----------|
| **Phi-3 Mini** | 3.8B params | Microsoft, CPU에서도 실행 가능 | CPU (16GB RAM) |
| **Gemma 2B** | 2B params | Google, 경량화, 빠른 응답 | CPU (8GB RAM) |
| **TinyLlama 1.1B** | 1.1B params | 초경량, 실험용 | CPU (4GB RAM) |

**장점**: 비용 0원, 데이터 프라이버시 보장
**단점**: 성능 제한적, 한국어 품질 낮음

#### 옵션 4: 다른 클라우드 LLM API
| 제공사 | 모델 | 장점 | 비용 |
|--------|------|------|------|
| **Anthropic** | Claude 3.5 Sonnet | 긴 컨텍스트 (200K tokens), 높은 품질 | ~$0.003/1K tokens |
| **Google** | Gemini 1.5 Pro | 멀티모달 지원, 무료 티어 있음 | ~$0.0025/1K tokens |
| **Upstage** | Solar LLM | 한국어 특화, 국내 서비스 | ~₩500/1K tokens |

#### 🔧 **구현 전략: LangChain으로 추상화**

```python
# config.py - 환경 변수로 모델 선택
LLM_PROVIDER = os.getenv("LLM_PROVIDER", "openai")  # openai, ollama, anthropic 등
LLM_MODEL = os.getenv("LLM_MODEL", "gpt-4-turbo")

# llm_factory.py - 팩토리 패턴으로 모델 생성
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic
from langchain_community.llms import Ollama

def get_llm(provider: str, model: str):
    if provider == "openai":
        return ChatOpenAI(model=model, temperature=0)
    elif provider == "anthropic":
        return ChatAnthropic(model=model, temperature=0)
    elif provider == "ollama":
        return Ollama(model=model)  # Llama3, Qwen 등
    elif provider == "gemini":
        from langchain_google_genai import ChatGoogleGenerativeAI
        return ChatGoogleGenerativeAI(model=model)
    else:
        raise ValueError(f"Unknown provider: {provider}")
```

**권장 로드맵**:
1. **Phase 1 (프로토타입)**: OpenAI GPT-4 (빠른 개발, 검증)
2. **Phase 2 (비용 최적화)**: Self-hosted Llama 3 8B (추론 서버 구축)
3. **Phase 3 (하이브리드)**: 간단한 질문은 SLM, 복잡한 질문은 GPT-4

### 📚 2.2 RAG (Retrieval-Augmented Generation) 스택

| 항목 | 기술 | 선정 이유 |
|------|------|----------|
| **RAG 프레임워크** | LangChain | 모델 독립적, RAG 파이프라인 구축 용이 |
| **Embedding 모델** | OpenAI `text-embedding-3-small` 또는 **BGE-M3** (Self-hosted) | 한국어 임베딩 품질 우수 |
| **Vector Database** | **Option 1**: Pinecone (클라우드) <br> **Option 2**: Weaviate (Self-hosted) <br> **Option 3**: pgvector (PostgreSQL extension) | Pinecone: 간편, 유료 <br> Weaviate: 무료, 도커 배포 <br> pgvector: 기존 DB 활용 |
| **문서 처리** | LangChain TextSplitter | 리뷰를 청크 단위로 분할 (512 tokens) |

**Embedding 모델 비교**:
| 모델 | 크기 | 비용 | 한국어 성능 |
|------|------|------|----------|
| OpenAI `text-embedding-3-small` | - | $0.00002/1K tokens | ⭐⭐⭐⭐⭐ |
| **BGE-M3** (Self-hosted) | 560M params | 무료 | ⭐⭐⭐⭐ |
| **KoSimCSE** (Self-hosted) | 110M params | 무료 | ⭐⭐⭐⭐ (한국어 특화) |

**Vector DB 선택 가이드**:
- **초기 프로토타입**: Pinecone (클라우드, 간편)
- **비용 절감**: pgvector (기존 PostgreSQL 활용)
- **확장성 필요**: Weaviate (Self-hosted, 무료)

## 3. RAG 기반 데이터 파이프라인

### 🔄 3.1 전체 아키텍처 (AI Agent 중심)

```
┌──────────────┐
│ 올리브영      │
│ (데이터 소스) │
└──────┬───────┘
       │
       ▼
┌──────────────────────────────────────────────────────┐
│ STEP 1: 크롤링 (Playwright)                          │
│ - 제품 검색 → 리뷰 수집                               │
│ - 출력: CSV (제품명, 날짜, 별점, 스킨타입, 리뷰내용)  │
└──────┬───────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────┐
│ STEP 2: 전처리 + 신뢰도 계산 (Pandas)                │
│ - 이모지 제거 → 오타 교정 → 품질 점수 계산           │
│ - ⭐ 신뢰도 점수 계산 (리뷰 길이, 구체성, 재구매 등)  │
│ - 출력: clean_text, quality_score, trust_score       │
└──────┬───────────────────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────────────────┐
│ STEP 3: 임베딩 생성 (LangChain + Embedding Model)    │
│ - 리뷰 텍스트 → 벡터 임베딩 (1536차원 또는 768차원)  │
│ - 청크 분할 (긴 리뷰는 512 tokens 단위로 분할)       │
│ - 메타데이터 추가 (제품명, 별점, 피부타입, 신뢰도)    │
└──────┬───────────────────────────────────────────────┘
       │
       ▼
```

```sql
CREATE TABLE reviews (
    id SERIAL PRIMARY KEY,
```

## 4. 핵심 기능 명세 (AI Agent 중심)

### 🤖 4.1 기능 1: AI 챗봇 (자연어 질의응답) - 핵심 기능

#### 4.1.1 개요
사용자가 자연어로 질문을 입력하면, RAG 기반으로 관련 리뷰를 검색하여 LLM이 답변을 생성합니다.

#### 4.1.2 지원 질의 유형

| 질의 유형 | 예시 질문 | RAG 동작 |
|---------|----------|---------|
| **제품 추천** | "건성 피부에 좋은 보습 크림 추천해줘" | 피부타입="건성" + 카테고리="보습" 필터링 → 유사 리뷰 검색 → GPT-4가 추천 |
| **제품 비교** | "라운드랩 자작나무 로션 vs 아누아 히알루론산 세럼 비교" | 두 제품 리뷰 각각 검색 → 10대 카테고리별 비교표 생성 |
| **특정 고민 해결** | "여드름 자국에 효과 있는 제품 알려줘" | 키워드="여드름", "자국", "흉터" 포함 리뷰 검색 → 효과 있는 제품 추천 |
| **가격대별 추천** | "2만원 이하 세럼 추천" | 가격 필터 + 별점 높은 순 정렬 |
| **성분 질문** | "나이아신아마이드 들어간 제품 알려줘" | 리뷰에서 "나이아신" 키워드 검색 → 해당 제품 추천 |
| **사용법 질문** | "토너 바른 후에 뭘 발라야 해?" | 리뷰에서 루틴 관련 내용 검색 → 단계별 추천 |

#### 4.1.3 입력

- **텍스트 입력창** (Google 검색 스타일)
  - 플레이스홀더: "피부 고민이나 궁금한 제품을 자유롭게 물어보세요"
  - 자동완성 기능 (선택적)
  - 음성 입력 지원 (선택적, Web Speech API)

#### 4.1.4 처리 로직 (RAG 파이프라인)

```python
# 의사 코드
def answer_question(user_question: str):
    # 1. 질문 임베딩 생성
    question_embedding = embedding_model.embed(user_question)

    # 2. Vector DB에서 유사 리뷰 검색
    similar_reviews = vector_db.search(
        embedding=question_embedding,
        limit=10,
        filter={
            "trust_score": {"$gte": 0.7},  # 신뢰도 70% 이상만
            # 추가 필터: 피부타입, 별점 등 (질문에서 추출)
        }
    )

    # 3. 프롬프트 생성
    context = "\n".join([f"- {r['product']}: {r['text']}" for r in similar_reviews])
    prompt = f"""
    당신은 화장품 추천 전문가입니다. 다음 리뷰들을 바탕으로 사용자 질문에 답변해주세요.

    [리뷰 데이터]
    {context}

    [사용자 질문]
    {user_question}

    [답변 형식]
    - 추천 제품 3개 (제품명, 추천 이유, 주의사항)
    - 각 제품의 장점과 단점
    - 신뢰도 높은 리뷰 내용 인용
    """

    # 4. LLM 호출 (스트리밍)
    response = llm.stream(prompt)

    # 5. 응답 반환 (Server-Sent Events)
    for chunk in response:
        yield chunk
```

```javascript
import * as d3 from 'd3';

function WordCloud({ words }) {
  useEffect(() => {
    const layout = cloud()
      .size([600, 400])
      .words(words.map(d => ({ text: d.text, size: d.value })))
      .padding(5)
      .rotate(() => 0)
      .font('Pretendard')
      .fontSize(d => d.size)
      .on('end', draw);

    function draw(words) {
      d3.select('#wordcloud')
        .append('svg')
        .attr('width', 600)
        .attr('height', 400)
        .append('g')
        .attr('transform', 'translate(300,200)')
        .selectAll('text')
        .data(words)
        .enter().append('text')
        .style('font-size', d => `${d.size}px`)
        .style('fill', d => getSentimentColor(d.sentiment))
        .attr('text-anchor', 'middle')
        .attr('transform', d => `translate(${d.x}, ${d.y})`)
        .text(d => d.text);
    }

    layout.start();
  }, [words]);

  return <div id="wordcloud"></div>;
}
```

## 6. 리뷰 신뢰도 평가 시스템

**(이미 섹션 4.4에 상세히 기술되어 있으므로 여기서는 요약)**

### 신뢰도 점수 산정 공식

```python
trust_score = (
    length_score * 0.20 +       # 리뷰 길이 (50자 미만 0점, 200자 이상 100점)
    specificity_score * 0.30 +  # 구체적 키워드 개수
    repurchase_score * 0.25 +   # 재구매 태그 여부
    usage_period_score * 0.15 + # 사용 기간 (한달이상 100점)
    skin_type_score * 0.10      # 피부타입 명시 여부
)
```

### 활용
- **RAG 검색**: 신뢰도 70% 이상만 사용
- **제품 카드**: 신뢰도 뱃지 표시
- **정렬**: 신뢰도 높은 리뷰 우선 표시

### 7.4 용어 정리

| 용어 | 설명 |
|------|------|
| **TF-IDF** | Term Frequency-Inverse Document Frequency. 문서 내 단어의 중요도를 계산하는 통계적 기법 |
| **형태소 분석** | 한글 텍스트를 명사, 동사, 형용사 등의 최소 의미 단위로 분해하는 기술 |
| **워드클라우드** | 단어의 빈도를 시각적 크기로 표현한 시각화 기법 |
| **시계열 데이터** | 시간 순서대로 정렬된 데이터 (예: 일별 리뷰 수) |
| **감성 분석** | 텍스트의 긍정/부정/중립 감정을 분류하는 NLP 기법 |
| **Celery** | Python 기반 비동기 태스크 큐 라이브러리 |
| **Redis** | In-memory 데이터 저장소, 캐싱 및 메시지 브로커로 사용 |
