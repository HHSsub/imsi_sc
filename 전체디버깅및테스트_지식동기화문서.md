## ✅ 기능 테스트 체크리스트

### 백엔드 API 테스트

#### 1. KPI API (`GET /api/kpi`)
- [ ] **정상 케이스**: `product=라운드랩` → 응답 200, JSON 형식 확인
- [ ] **데이터 정확성**: `total_reviews`, `avg_rating`, `positive_ratio`, `negative_ratio` 값 검증
- [ ] **존재하지 않는 제품**: `product=존재하지않는제품` → 404 에러
- [ ] **필수 파라미터 누락**: `product` 없이 호출 → 422 에러
- [ ] **한글 인코딩**: `product=아누아` → 정상 동작
- [ ] **CORS 헤더**: 응답 헤더에 `Access-Control-Allow-Origin` 포함

**테스트 명령**:
```bash
curl "http://localhost:8000/api/kpi?product=라운드랩"
curl "http://localhost:8000/api/kpi?product=존재하지않는제품"
curl "http://localhost:8000/api/kpi"  # 파라미터 누락
```

---

#### 2. 시계열 API (`GET /api/timeseries`)
- [ ] **정상 케이스**: `product=라운드랩` → 응답 200
- [ ] **데이터 형식**: `dates`, `positive_counts`, `negative_counts`, `avg_ratings` 배열 확인
- [ ] **배열 길이 일치**: 4개 배열 길이가 모두 동일
- [ ] **날짜 정렬**: `dates` 배열이 오름차순 정렬
- [ ] **날짜 형식**: ISO 8601 (YYYY-MM-DD)
- [ ] **존재하지 않는 제품**: 404 에러

**테스트 명령**:
```bash
curl "http://localhost:8000/api/timeseries?product=라운드랩"
```

---

#### 3. 긍정 키워드 API (`GET /api/keywords/positive`)
- [ ] **정상 케이스**: `product=라운드랩&limit=10` → 응답 200
- [ ] **데이터 형식**: `keywords` 배열 확인, 각 항목에 `word`, `count` 포함
- [ ] **정렬**: `count` 내림차순 정렬
- [ ] **개수 제한**: `limit=10` → 최대 10개 반환
- [ ] **기본값**: `limit` 없이 호출 → 30개 반환 (기본값)
- [ ] **존재하지 않는 제품**: 404 에러

**테스트 명령**:
```bash
curl "http://localhost:8000/api/keywords/positive?product=라운드랩&limit=10"
curl "http://localhost:8000/api/keywords/positive?product=라운드랩"  # limit 기본값
```

---

#### 4. 부정 키워드 API (`GET /api/keywords/negative`)
- [ ] **정상 케이스**: `product=라운드랩&limit=10` → 응답 200
- [ ] **데이터 형식**: 긍정 키워드와 동일
- [ ] **정렬**: `count` 내림차순
- [ ] **개수 제한**: `limit` 적용
- [ ] **긍정과 다른 키워드**: 긍정 키워드와 중복되지 않음 확인

**테스트 명령**:
```bash
curl "http://localhost:8000/api/keywords/negative?product=라운드랩&limit=10"
```

### BUG-001: /api/kpi 응답 시 avg_rating null

**발견일**: 2026-02-17
**심각도**: 높음
**분류**: Backend
**담당자**: Backend Agent

**설명**:
`/api/kpi` API 호출 시 `avg_rating` 필드가 null로 반환됨

**재현 시나리오**:
1. 백엔드 서버 실행: `uvicorn app.main:app --reload`
2. curl 호출: `curl "http://localhost:8000/api/kpi?product=라운드랩"`
3. 응답 확인

**기대 결과**:
```json
{
  "total_reviews": 7305,
  "avg_rating": 4.2,
  "positive_ratio": 0.78,
  "negative_ratio": 0.12
}
```

```json
{
  "total_reviews": 7305,
  "avg_rating": null,
  "positive_ratio": 0.78,
  "negative_ratio": 0.12
}
```

**원인 추정**:
- DB에 `rating` 데이터가 없거나
- 평균 계산 로직 오류

**우선순위**: P0 (즉시)

**상태**: 🔴 Open → 🟡 In Progress → 🟢 Closed (수정 후)

### 3. 수정 완료 후 반드시 재테스트
- 버그 수정 후 → QA Agent가 재테스트
- 통과 시 → 상태 "Closed"로 변경
- 실패 시 → 다시 "In Progress"로 변경, 추가 정보 기록

---

### 4. 심각도 기준
- **높음 (High)**: 기능 완전 중단, 데이터 손실
- **중간 (Medium)**: 기능 일부 오류, 사용 가능하지만 불편
- **낮음 (Low)**: UI 오류, 오타, 스타일 문제

---

### 5. 우선순위 기준
- **P0 (즉시)**: 릴리스 차단, 즉시 수정 필요
- **P1 (높음)**: 릴리스 전 수정 필요
- **P2 (보통)**: 다음 릴리스에서 수정
- **P3 (낮음)**: 여유 있을 때 수정
